Message class

This class at the base represents an abstraction representation of a message that needs to be sent to the server. As one of the core principles of the SimplyReliable API, the Message class abstracts out the concept of a HTTP Method. For each of the HTTP Methods, the Message class has the appropriate subclass implementation defined specifically for the particular HTTP Method.

Message Hierarchy

PayLoadMessage class

Represents all the Messages that will have data in the request to be sent to the server. The API will send the data in JSON format, in the request body. The API user just needs to be pass any object to the PayLoadMessage, and the API will convert the object to the appropriate JSON representation. (It uses the GSON library for the conversion). Also the object is required to be Serializable as the API will internally store the serialized object in its internal data store.

Based on the REST semantics and the HTTP method of the call, the user would either create a PostMessage (a new resource getting created at the server side) or a PutMessage (an existing resource getting updated at the server side). Both of them are subclasses of the PayLoadMessage class.

The minimum properties required by a PayLoadMessage is :

  • Resource
  • Serializable object representing the request data
  • Type object representing the object type to which the response JSON should be converted to

GetMessage class

Represents all the Messages that will ask for data from the server. A GetMessage cannot send any data in the request to the server. The only data that can be sent to the server in the GetMessage request, is through QueryParams.

The minimum properties required by a GetMessage is :

  • Resource
  • Type object representing the object type to which the response JSON should be converted to

MultiPartGetMessage class

A special subclass of GetMessage. Used to download files from the server.

The minimum properties required by a MultiPartGetMessage is :

  • Resource
  • Local file path(till the name of the file) where the downloaded file must be saved

DeleteMessage class

Represents a Message that is sent to the server to delete an existing resource from the server. A DeleteMessage cannot send any data in the request body to the server. The only data that can be sent to the server is the resource id as path params in the Resource path.

The minimum properties required by a DeleteMessage is :

  • Resource
