REST Semantics

The SimplyReliable API internally uses REST as the web service architectural style for making network calls to the server. The server implementation needs to be REST based, in order to be used by the api.

Http Methods

The internal REST based implementation of the api, supports the following Http Methods (Called as Verbs, in a REST terminology).
Note : The format of data exchange fixed by the api is JSON :

  • GET : Used to 'get' data (resources in REST terminology) from the server. No data sent in the request. Data is received from the server in JSON format.
  • POST : Used to 'post' data to the server. This methos usually creates a new data (resource in REST terminology) at the server side. Data is sent in the request in a JSON format. The response sent back by the server is usually all the fields that were sent in the request for a resource, along with the 'id' (id of the newly created resource at the server side) and other fields set.
    Note : Since it is a mobile environment where bandwidth is sacred, avoid sending redundant data in the response. Ideally send only the 'id' of the newly created resource and other extra fields if required at the client side.
