SimplyReliableInfra class

One of the core and the most important classes of the API. This class is responsible for setting up the infrastructure and initialization required by the API. Some of the common functions of this class is to help the API user to build new messages, sending messages, setting the custom HTTP headers, sending infrastructure lifecycle events to all, to name a few.

Initializing the API

The first thing to do before creating messages and sending to the server, is to initialize the SimplyReliable API.

SimplyReliableInfra.init(ROOT_URL, customHeadersIfAny, applicationContextObject);

Note: In an Android environment, the Application class onCreate() qualifies to be the best place for initializing the API.

Singleton Instance

The SimplyReliableInfra object is implemented in the API as a singleton instance. In order to get a reference of this singleton instance :

SimplyReliableInfra obj = SimplyReliableInfra.getInstance();

Note: Make sure to call init() before asking for this singleton object.

Creating Different Message Types

The SimplyReliableInfra object can be used to create Message instances of different types, using the create<<message-type>>OutMessage() method family.

SimplyReliableInfra obj = SimplyReliableInfra.getInstance();
OutMessage postMessage = obj.createPostOutMessage(variousArguments);

The above code creates a PostMessage. Similarly other message types can be created using the appropriate methods.

SimplyReliableInfra obj = SimplyReliableInfra.getInstance();
OutMessage postMessage = obj.createPostOutMessage(initialArguments, callbackObj);

The above code creates a PostMessage that is asynchronous in nature and can be used in offline conditions too.

SimplyReliableInfra obj = SimplyReliableInfra.getInstance();
OutMessage postMessage = obj.createPostOutMessage(argumentsWithoutAnyCallbackObjPassed);

The above code creates a PostMessage that is synchronous in nature and can be used only in online conditions only.

Search Messages by Callback type

We can search for asynchronous Messages on the basis of their callback type, from the infrastructure using the following :

List<OutMessage> messages = simplyReliableInfraObj.getExistingMessagesByCallbackType(callbackType)

SimplyReliableInfraIntf Interface

Interface that defines all the methods that can be called at a specific point in the lifecycle of the SimplyReliable API. The SimplyReliableInfra object stores the observer list for the SimplyReliableInfraIntf.
Object wishing to become observer for the events from the SimplyReliableInfra object must implement the SimplyReliableInfraIntf interface and call the following

simplyReliableInfraObj.attach(objectWishingToBecomeObserver)

Note:The SimplyReliableInfraIntf observer must remove itself from the observer list of the SimplyReliableInfra object, once its work is done.

simplyReliableInfraObj.detach(observerWishingToRemoveItselfFromObserverList)

From the SimplyReliableInfra subject side, use the publishXXXX() family, for publishing SimplyReliableInfra object lifecycle methods to all its observers.

simplyReliableInfraObj.publishServerDown()

The above publishes the 'server down' event to all its observers.
